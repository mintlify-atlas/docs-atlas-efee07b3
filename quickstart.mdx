# Quick Start Guide

This guide will walk you through creating and running your first Underpost application. You'll have a fully functional PWA/API-REST project running locally in just a few minutes.

## Prerequisites

Before you begin, ensure you have the following installed:

- Node.js v24.10.0 or higher
- npm v11.6.2 or higher
- Git

<Note>
For detailed system requirements and installation instructions, see the [Installation](/installation) page.
</Note>

## Installation

<Steps>
  <Step title="Install Underpost CLI">
    Install the Underpost framework globally to access the command-line tools:
    
    ```bash
    npm install -g underpost
    ```
    
    Verify the installation:
    
    ```bash
    underpost --version
    ```
    
    You should see version `3.0.2` or higher.
  </Step>

  <Step title="Create a New Project">
    Generate a new Underpost project using the scaffolding command:
    
    ```bash
    underpost new my-app
    ```
    
    This command will:
    - Create a new directory called `my-app`
    - Set up the project template with all necessary files
    - Install dependencies
    - Start the development server automatically
    
    <Note>
    The server will automatically start running on **http://localhost:4001** after template installation.
    </Note>
  </Step>

  <Step title="Navigate to Your Project">
    Open your browser and visit:
    
    ```
    http://localhost:4001
    ```
    
    You should see your new Underpost application running!
  </Step>
</Steps>

## Project Structure

After creating your project, you'll have the following structure:

```
my-app/
├── src/
│   ├── client/          # Client-side PWA code
│   ├── server/          # Server-side API code
│   └── index.js         # Main entry point
├── .env.development     # Development environment variables
├── .env.production      # Production environment variables
├── package.json         # Project dependencies
└── ...
```

## Development Workflow

### Building the Client Bundle

To build the client-side application for production:

```bash
cd my-app
npm run build
```

This compiles and optimizes your PWA assets.

### Running the Development Server

To start the development server with hot reloading:

```bash
npm run dev
```

The server will watch for file changes and automatically reload.

### Running Different Components

<CodeGroup>

```bash API Server Only
npm run dev-api
```

```bash Client Only
npm run dev-client
```

```bash Production Mode
npm start
```

</CodeGroup>

## Available Commands

Your Underpost project comes with several npm scripts:

| Command | Description |
|---------|-------------|
| `npm start` | Run the production server |
| `npm run dev` | Start development server with hot reload |
| `npm run build` | Build the client bundle for production |
| `npm run dev-api` | Run API server only (development) |
| `npm run dev-client` | Run client development server only |
| `npm test` | Run the test suite |
| `npm run docs` | Generate JSDoc documentation |

## Troubleshooting

<AccordionGroup>
  <Accordion title="Server won't start on port 4001">
    The port might be in use. Check if another process is using port 4001:
    ```bash
    lsof -i :4001
    ```
    You can modify the port in your `.env.development` file.
  </Accordion>
  
  <Accordion title="npm install fails">
    Ensure you have the correct Node.js and npm versions:
    ```bash
    node --version  # Should be v24.10.0 or higher
    npm --version   # Should be v11.6.2 or higher
    ```
  </Accordion>
  
  <Accordion title="Build errors">
    Try clearing the cache and reinstalling dependencies:
    ```bash
    rm -rf node_modules package-lock.json
    npm install
    ```
  </Accordion>
</AccordionGroup>
