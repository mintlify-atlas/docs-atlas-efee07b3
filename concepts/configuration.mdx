The Underpost framework uses a centralized configuration object called `DefaultConf` that defines all aspects of your application. Configuration is organized into four main sections: client, server, SSR, and cron.

## Configuration Structure

```javascript
const DefaultConf = {
  client: { /* Client configuration */ },
  server: { /* Server configuration */ },
  ssr: { /* SSR configuration */ },
  cron: { /* Scheduled jobs */ }
};
```

## Client Configuration

The client configuration defines the frontend application setup.

### Metadata

Define your PWA metadata:

```javascript
client: {
  default: {
    metadata: {
      title: 'PWA Demo App',
      backgroundImage: 'assets/background/white0-min.jpg',
      description: 'Web application',
      keywords: ['web', 'app', 'spa', 'demo', 'github-pages'],
      author: 'https://github.com/underpostnet',
      thumbnail: 'android-chrome-384x384.png',
      themeColor: '#ececec',
      pwaAssetsPath: '',
    }
  }
}
```

### Component Loading

Specify which components to load:

```javascript
components: {
  core: [
    'CommonJs',
    'VanillaJs',
    'Responsive',
    'Router',
    'Modal',
    'SocketIo',
    'Logger',
    'Auth',
    // ... more core components
  ],
  default: [
    'MenuDefault',
    'RoutesDefault',
    'LogInDefault',
    'TranslateDefault',
    // ... custom components
  ]
}
```

<Note>
Core components provide framework functionality, while custom components implement your application logic.
</Note>

### Views and Routes

Define your application views:

```javascript
views: [
  { path: '/', title: 'Home', client: 'Default', ssr: 'Default' },
  { path: '/home', title: 'Home', client: 'Default', ssr: 'Default' },
  { path: '/settings', client: 'Default', ssr: 'Default' },
  { path: '/log-in', client: 'Default', ssr: 'Default' },
  { path: '/sign-up', client: 'Default', ssr: 'Default' },
  { path: '/account', client: 'Default', ssr: 'Default' },
  { path: '/docs', client: 'Default', ssr: 'Default' },
  { path: '/u', client: 'Default', ssr: 'Default' },
  { path: '/404', title: '404 Not Found', client: 'Default', ssr: 'Default' },
  { path: '/500', title: '500 Server Error', client: 'Default', ssr: 'Default' },
]
```

### External Dependencies

Configure third-party library distributions:

```javascript
dists: [
  {
    folder: './node_modules/@neodrag/vanilla/dist/min',
    public_folder: '/dist/@neodrag-vanilla',
    import_name: '@neodrag/vanilla',
    import_name_build: '/dist/@neodrag-vanilla/index.js',
  },
  {
    folder: './node_modules/socket.io/client-dist',
    public_folder: '/dist/socket.io',
    import_name: 'socket.io/client-dist/socket.io.esm.min.js',
    import_name_build: '/dist/socket.io/socket.io.esm.min.js',
  },
  // ... more dependencies
]
```

### Services

List the API services your client will use:

```javascript
services: ['default', 'core', 'user', 'test', 'file', 'document']
```

## Server Configuration

The server configuration maps hosts and paths to runtime settings.

### Host Configuration

```javascript
server: {
  'default.net': {
    '/': {
      client: 'default',
      runtime: 'nodejs',
      apis: ['default', 'core', 'user', 'test', 'file', 'document'],
      origins: [],
      minifyBuild: false,
      iconsBuild: false,
      liteBuild: true,
      docsBuild: false,
      offlineBuild: false,
      ws: 'core',
      peer: true,
      proxy: [80, 443],
      db: {
        provider: 'mongoose',
        host: 'mongodb://127.0.0.1:27017',
        name: 'default'
      },
      mailer: {
        sender: { email: 'noreply@default.net', name: 'Default' },
        transport: {
          host: 'smtp.default.com',
          port: 465,
          secure: true,
          auth: { user: 'noreply@default.net', pass: '' }
        }
      },
      valkey: {
        port: 6379,
        host: '127.0.0.1',
      }
    }
  }
}
```

### Configuration Options

<ParamField path="client" type="string" required>
  The client configuration ID to use for this path
</ParamField>

<ParamField path="runtime" type="string" default="nodejs">
  The server runtime environment
</ParamField>

<ParamField path="apis" type="array">
  List of API modules to load for this path
</ParamField>

<ParamField path="ws" type="string">
  WebSocket handler ID for real-time features
</ParamField>

<ParamField path="peer" type="boolean" default={false}>
  Enable PeerJS server on `/peer` path
</ParamField>

<ParamField path="proxy" type="array">
  Port numbers for proxy configuration (e.g., [80, 443])
</ParamField>

<ParamField path="db" type="object">
  Database configuration with provider, host, and name
</ParamField>

<ParamField path="valkey" type="object">
  Valkey/Redis configuration for caching
</ParamField>

```javascript
    offline: [
      {
        path: '/offline',

        client: 'NoNetworkConnection',
        head: [],
        body: []
      },
      {
        path: '/maintenance',
        title: 'Server Maintenance',
        client: 'Maintenance',
        head: [],
        body: []
      }
    ],
    pages: [
      {
        path: '/test',
        title: 'Test',
        client: 'Test',
        head: [],
        body: []
      }
    ]
  }

```

## Cron Configuration

Schedule automated tasks:

```javascript
cron: {
  records: {
    A: [
      {
        host: 'example.com',
        dns: 'dondominio',
        api_key: '???',
        user: '???'
      }
    ]
  },
  jobs: {
    dns: {
      expression: '* * * * *',
      enabled: true,
      instances: 1
    },
    backups: {
      expression: '0 1 * * *',
      enabled: true,
      instances: 1
    }
  }
}
```

<Warning>
Cron expressions use standard cron syntax. The `dns` job runs every minute, while `backups` runs daily at 1 AM.
</Warning>

## Loading Configuration

The framework automatically loads configuration based on deployment ID:

```javascript
import { Config } from './src/server/conf.js';

// Access default configuration
const conf = Config.default;

// Build configuration for specific deployment
await Config.build('dd-production', deployList, subConf);
```

## Environment-Specific Configuration

Configuration files are stored per deployment:

```
engine-private/conf/dd-default/
├── conf.client.json
├── conf.server.json
├── conf.ssr.json
├── conf.cron.json
├── .env.production
├── .env.development
└── .env.test
```

<Tip>
Use sub-configurations for development variations:
- `conf.server.dev.local.json`
- `conf.server.dev.staging.json`
</Tip>
