---
title: underpost metadata
description: Manage cluster metadata operations including import, export, and generation
---

The `underpost metadata` command manages cluster metadata operations, providing tools to import, export, and generate metadata for deployments, cron jobs, and instances.

## Usage

```bash
underpost metadata [deploy-id] [host] [path] [options]
```

## Arguments

<ParamField path="deploy-id" type="string" optional>
  The deployment ID to manage metadata for.

  ```bash
  underpost metadata dd-default
  ```
</ParamField>

<ParamField path="host" type="string" optional>
  The host to manage metadata for.

  ```bash
  underpost metadata dd-default example.com
  ```
</ParamField>

<ParamField path="path" type="string" optional>
  The path to manage metadata for.

  ```bash
  underpost metadata dd-default example.com /api
  ```
</ParamField>

## Options

### Operation Options

<ParamField path="--import" type="boolean">
  Imports metadata from local storage.

  ```bash
  underpost metadata dd-default --import
  ```
</ParamField>

<ParamField path="--export" type="boolean">
  Exports metadata to local storage.

  ```bash
  underpost metadata dd-default --export
  ```
</ParamField>

<ParamField path="--generate" type="boolean">
  Generates cluster metadata for the specified deployment.

  ```bash
  underpost metadata dd-default --generate
  ```
</ParamField>

### Data Collection Options

<ParamField path="--crons" type="boolean">
  Apply operation to cron data collection.

  ```bash
  underpost metadata dd-default --export --crons
  ```
</ParamField>

<ParamField path="--instances" type="boolean">
  Apply operation to instance data collection.

  ```bash
  underpost metadata dd-default --import --instances
  ```
</ParamField>

### Execution Context

<ParamField path="--itc" type="boolean">
  Apply operation under container execution context.

  ```bash
  underpost metadata dd-default --export --itc
  ```
</ParamField>

## Use Cases

### Backup and Restore

Regularly export metadata for backup:

```bash
# Daily backup script
#!/bin/bash
DATE=$(date +%Y%m%d)
underpost metadata dd-production --export
cp -r ./metadata-backup ./backups/metadata-$DATE
```

### Migration Between Clusters

```bash
# Export from source cluster
underpost metadata dd-source --export --crons --instances

# Copy to destination
scp -r ./metadata-backup user@dest-server:/path/to/metadata

# Import on destination cluster
underpost metadata dd-dest --import --crons --instances
```

### Configuration Versioning

```bash
# Export current configuration
underpost metadata dd-default --export

# Commit to version control
git add metadata-backup/
git commit -m "Update cluster metadata"
git push
```

### Testing and Development

```bash
# Generate test metadata
underpost metadata dd-dev --generate

# Export for sharing
underpost metadata dd-dev --export

# Share with team
tar -czf dev-metadata.tar.gz metadata-backup/
```

## Metadata Storage

Metadata is typically stored in:

```
./metadata-backup/
├── crons/
│   ├── dd-default-cron-backup.json
│   └── dd-default-cron-jobs.json
└── instances/
    ├── dd-default-instances.json
    └── dd-default-instance-configs.json
```

## Troubleshooting

### Import Fails

If import fails, verify metadata files exist:

```bash
ls -la metadata-backup/
```

### Export Path Issues

Ensure write permissions:

```bash
chmod 755 ./metadata-backup
```

### Container Context Issues

When using `--itc`, ensure container is running:

```bash
kubectl get pods | grep dd-default
```
