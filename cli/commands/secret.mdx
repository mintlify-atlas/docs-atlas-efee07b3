The `underpost secret` command manages secrets for various platforms, providing secure storage and retrieval of sensitive configuration data.

## Usage

```bash
underpost secret <platform> [options]
```

## Arguments

<ParamField path="platform" type="string" required>
  The secret management platform. Currently supported: `underpost`.

  ```bash
  underpost secret underpost --init
  ```
</ParamField>

## Options

<ParamField path="--init" type="boolean">
  Initializes the secrets platform environment.

  ```bash
  underpost secret underpost --init
  ```
</ParamField>

<ParamField path="--create-from-file" type="string">
  Creates secrets from a specified environment file. Provide the path to the `.env` file.

  ```bash
  underpost secret underpost --create-from-file .env.production
  ```
</ParamField>

<ParamField path="--list" type="boolean">
  Lists all available secrets for the platform.

  ```bash
  underpost secret underpost --list
  ```
</ParamField>

## Examples

### Initialize Secret Management

```bash
underpost secret underpost --init
```

### Create Secrets from Environment File

```bash
underpost secret underpost --create-from-file ./engine-private/conf/dd-default/.env.production
```

This reads all key-value pairs from the environment file and stores them securely in Underpost's configuration system.

### List All Secrets

```bash
underpost secret underpost --list
```

### Secret Management Workflow

<Steps>
  <Step title="Initialize secret management">
    ```bash
    underpost secret underpost --init
    ```
  </Step>
  
  <Step title="Prepare environment file">
    Create a `.env` file with your secrets:
    ```bash
    # .env.production
    DATABASE_URL=postgresql://user:pass@localhost:5432/db
    API_KEY=your-secret-api-key
    JWT_SECRET=your-jwt-secret
    ```
  </Step>
  
  <Step title="Import secrets from file">
    ```bash
    underpost secret underpost --create-from-file .env.production
    ```
  </Step>
  
  <Step title="Verify secrets were imported">
    ```bash
    underpost secret underpost --list
    ```
  </Step>
</Steps>

### Development vs Production Secrets

```bash
# Import development secrets
underpost secret underpost --create-from-file .env.development

# Import production secrets
underpost secret underpost --create-from-file .env.production
```

### Deploy-Specific Secrets

```bash
# Import secrets for specific deployment
underpost secret underpost --create-from-file ./engine-private/conf/dd-api/.env.production
```

## Environment File Format

The environment file should follow standard `.env` format:

```bash
# Database configuration
DATABASE_URL=postgresql://user:password@localhost:5432/mydb
DB_POOL_SIZE=10

# API Keys
API_KEY=your-api-key-here
SECRET_KEY=your-secret-key-here

# Authentication
JWT_SECRET=your-jwt-secret
JWT_EXPIRATION=3600

# External Services
STRIPE_API_KEY=sk_test_...
SENDGRID_API_KEY=SG...

# Application Settings
NODE_ENV=production
PORT=3000
```

## Security Best Practices

### Never Commit Secrets

Add environment files to `.gitignore`:

```bash
# .gitignore
.env
.env.*
*.env
engine-private/conf/**/.env.*
```

### Use Different Secrets per Environment

```bash
# Development
underpost secret underpost --create-from-file .env.development

# Staging
underpost secret underpost --create-from-file .env.staging

# Production
underpost secret underpost --create-from-file .env.production
```

### Rotate Secrets Regularly

```bash
# Update environment file with new secrets
vim .env.production

# Re-import updated secrets
underpost secret underpost --create-from-file .env.production
```

### Access Control

Restrict access to secret files:

```bash
chmod 600 .env.production
chown your-user:your-group .env.production
```

## Integration with Deployments

Secrets are automatically available in deployment containers:

```bash
# Secrets are loaded during deployment
underpost start dd-default production
```

The deployment process automatically:
1. Loads secrets from Underpost configuration
2. Makes them available as environment variables
3. Injects them into running containers

## Kubernetes Integration

When deploying to Kubernetes, secrets are managed via ConfigMaps:

```bash
# Secrets are automatically included in ConfigMaps
underpost deploy dd-default production --build-manifest
```
