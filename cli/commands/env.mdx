## Overview

The `underpost env` command manages environment variables and configurations related to specific deployment IDs. It loads configuration files and sets up the environment context for running services.

## Usage

```bash
underpost env [deploy-id] [env] [subConf]
```

## Arguments

<ParamField path="deploy-id" type="string" optional>
  The deployment configuration ID. Special values:
  - `clean` - Restore default environment settings
  - `root` - Load underpost root environment
  - `current` - Get plain current deploy ID
  - Custom deploy ID - Load configuration for that deployment
</ParamField>

<ParamField path="env" type="string" optional default="production">
  The environment to set (e.g., "production", "development")
</ParamField>

<ParamField path="subConf" type="string" optional>
  The sub-configuration to set
</ParamField>

## How It Works

When you run `underpost env`, the command:

<Steps>
  <Step title="Check for environment file">
    Looks for `./engine-private/conf/[deploy-id]/.env.[env]` file
  </Step>
  <Step title="Load configuration">
    If the file exists, loads it using dotenv with `override: true`
  </Step>
  <Step title="Apply configuration">
    Calls `loadConf(deployId, subConf)` to apply the full configuration context
  </Step>
</Steps>

## Examples

### Load production environment for a deployment

```bash
underpost env dd-default production
```

### Load development environment

```bash
underpost env dd-default development
```

### Load root environment

```bash
underpost env root
```

### Get current deploy ID

```bash
underpost env current
```

### Clean environment settings

```bash
underpost env clean
```

## Environment File Structure

The command loads files from:

```
./engine-private/conf/[deploy-id]/.env.[env]
```

Example structure:
```
engine-private/
└── conf/
    └── dd-default/
        ├── .env.production
        ├── .env.development
        └── .env.test
```
