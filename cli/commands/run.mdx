## Overview

The `underpost run` command provides 56 specialized runners for executing workflow scripts ranging from cluster management to deployment automation. Each runner encapsulates a specific workflow pattern.

## Usage

```bash
underpost run <runner-id> [path] [options]
```

## Arguments

<ParamField path="runner-id" type="string" required>

</ParamField>

<ParamField path="path" type="string" optional>
  Input value, identifier, or path specific to the runner operation
</ParamField>

## Available Runners

### Cluster Management

#### dev-cluster
Reset and deploy full development cluster with MongoDB, Valkey, and service exposure.

```bash
underpost run dev-cluster --dev --namespace default
```

#### cluster-build
Build configuration for cluster deployment.

```bash
underpost run cluster-build
```

### Deployment Operations

#### sync
Deploy with blue/green traffic switching and health monitoring.

```bash
underpost run sync dd-default,2,green,custom-image:latest,node-name --namespace production
```

Path format: `deployId,replicas,versions,image,node`

#### stop
Stop a deployment by deleting Kubernetes resources.

```bash
underpost run stop --deploy-id dd-default --namespace default
```

#### promote
Switch traffic between blue/green deployments.

```bash
underpost run promote dd-default,production,2 --namespace default
```

Path format: `deployId,env,replicas`

#### instance
Deploy instance from conf.instances.json configuration.

```bash
underpost run instance dd-default,web-1,3 --namespace default --tls
```

Path format: `deployId,instanceId,replicas`

#### instance-promote
Promote instance deployment traffic.

```bash
underpost run instance-promote dd-default,web-1 --namespace default
```

### SSH Remote Operations

#### ssh-deploy
Trigger SSH-based CD pipeline by committing and pushing.

```bash
underpost run ssh-deploy deploy-marker
```

#### ssh-deploy-info
Retrieve deployment status from remote server via SSH.

```bash
underpost run ssh-deploy-info dd --deploy-id dd-default --user root
```

#### ssh-deploy-stop
Stop remote deployment via SSH.

```bash
underpost run ssh-deploy-stop current --deploy-id dd-default --user root --namespace default
```

#### ssh-deploy-db
Import/restore database on remote deployment.

```bash
underpost run ssh-deploy-db dd-default --deploy-id dd-default --user root --namespace default
```

#### ssh-deploy-db-rollback
Perform database rollback via SSH.

```bash
underpost run ssh-deploy-db-rollback dd-default,3 --deploy-id dd-default --user root
```

Path format: `deployId,nCommits`

#### ssh-deploy-db-status
Check database statistics for deployments.

```bash
underpost run ssh-deploy-db-status dd --deploy-id dd-default --user root
```


#### release-deploy

```bash
underpost run release-deploy
```

## Examples

### Deploy with blue-green switching

```bash
# Deploy to green version with 3 replicas
underpost run sync dd-default,3,green --namespace production

# Promote green to receive traffic
underpost run promote dd-default,production,3 --namespace production
```

### Remote deployment via SSH

```bash
# Configure SSH credentials
underpost ssh --user-add --user deploy --deploy-id dd-default

# Deploy remotely
underpost run ssh-deploy production --deploy-id dd-default --user deploy

# Check status
underpost run ssh-deploy-info --deploy-id dd-default --user deploy
```

### Development workflow

```bash
# Setup dev cluster
underpost run dev-cluster --dev --namespace default

# Deploy with local hosts
underpost run dev-hosts-expose dd-default

# Run tests in container
underpost run dd-container "npm test" --dev

# Clean up
underpost run dev-hosts-restore
```

### Database operations

```bash
# Import database
underpost run ssh-deploy-db dd-default --deploy-id dd-default --user root

# Check database stats
underpost run ssh-deploy-db-status dd --deploy-id dd-default --user root

# Rollback 3 commits
underpost run ssh-deploy-db-rollback dd-default,3 --deploy-id dd-default --user root
```
