---
title: Test API
description: Utility endpoints for validation and testing
---

The Test API provides utility endpoints for validating emails, extracting YouTube IDs, and password strength checking.

## Authentication

- `POST /` requires authentication
- All other endpoints are public

## Endpoints

### Verify Email

Validates if a string is a valid email address.

<CodeGroup>
```bash cURL
curl -X GET "https://api.example.com/test/verify-email?email=user@example.com"
```

```javascript JavaScript
const response = await fetch(
  'https://api.example.com/test/verify-email?email=user@example.com'
);
const result = await response.json();
```
</CodeGroup>

#### Query Parameters

<ParamField query="email" type="string" required>
  The email address to validate
</ParamField>

#### Response

<ResponseField name="status" type="string">
  Response status: `success` or `error`
</ResponseField>

<ResponseField name="data" type="boolean">
  `true` if email is valid, `false` otherwise
</ResponseField>

<ResponseExample>
```json Success Response
{
  "status": "success",
  "data": true
}
```
</ResponseExample>

### Extract YouTube ID

Extracts the video ID from a YouTube URL.

<CodeGroup>
```bash cURL
curl -X GET "https://api.example.com/test/youtube-id?url=https://www.youtube.com/watch?v=dQw4w9WgXcQ"
```

```javascript JavaScript
const response = await fetch(
  'https://api.example.com/test/youtube-id?url=https://www.youtube.com/watch?v=dQw4w9WgXcQ'
);
const result = await response.json();
```
</CodeGroup>

#### Query Parameters

<ParamField query="url" type="string" required>
  The YouTube URL to parse
</ParamField>

#### Response

<ResponseField name="status" type="string">
  Response status: `success` or `error`
</ResponseField>

<ResponseField name="data" type="string">
  The extracted YouTube video ID
</ResponseField>

<ResponseExample>
```json Success Response
{
  "status": "success",
  "data": "dQw4w9WgXcQ"
}
```
</ResponseExample>

</CodeGroup>

#### Query Parameters

<ParamField query="password" type="string" required>

#### Response

<ResponseField name="status" type="string">
  Response status: `success` or `error`
</ResponseField>

<ResponseField name="data" type="boolean">

## Error Handling

All endpoints return a consistent error format:

```json
{
  "status": "error",
  "message": "Error description"
}
```

HTTP status code for errors: `400`
