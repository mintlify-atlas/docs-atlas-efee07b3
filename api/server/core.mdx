The Core API provides fundamental Create, Read, Update, and Delete operations for core database resources. All endpoints require admin authentication.

## Authentication

All Core API endpoints require:
- Valid JWT token in Authorization header
- Admin role (`adminGuard` middleware)

```bash
Authorization: Bearer <admin-jwt-token>
```

## Endpoints

### Create Resource

Create a new core resource.

```http
POST /api/core
POST /api/core/:id
```

<ParamField path="id" type="string" optional>
  Optional resource identifier
</ParamField>

<ParamField body="data" type="object" required>
  Resource data to create
</ParamField>

#### Example Request

```javascript
const response = await fetch('/api/core', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${adminToken}`,
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    // Your resource data
  })
});

const result = await response.json();
```

#### Response

<ResponseField name="status" type="string">
  Response status ("success" or "error")
</ResponseField>

<ResponseField name="data" type="object">
  Created resource object
</ResponseField>

```json
{
  "status": "success",
  "data": {
    "_id": "507f1f77bcf86cd799439011"
  }
}
```

## Error Handling

All endpoints return a 400 status code on error:

```json
{
  "status": "error",
  "message": "Error description"
}
```

## Implementation Details

The Core API uses a standard service pattern:

**Router** (`src/api/core/core.router.js:11-18`)
- Express Router with admin guard middleware
- Supports both `/` and `/:id` patterns for all methods

**Controller** (`src/api/core/core.controller.js:6-67`)
- Handles request/response transformation
- Returns standardized JSON responses
- Logs errors with stack traces

**Service** (`src/api/core/core.service.js:7-28`)
- Performs database operations using Mongoose
- Methods: `post`, `get`, `put`, `delete`

**Model** (`src/api/core/core.model.js:5`)
- Empty Mongoose schema (base implementation)
- Extend as needed for your use case
