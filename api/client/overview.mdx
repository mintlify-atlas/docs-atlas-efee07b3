## Introduction

The Underpost client framework provides a comprehensive set of components organized into three main categories:

- **Core Components** - Foundation modules for DOM manipulation, routing, modals, and authentication
- **UI Components** - Reusable interface elements like buttons, inputs, dropdowns, and grids
- **Services** - Utility services for translation, logging, and validation

All components are built with vanilla JavaScript and designed to work together seamlessly.

## Component Architecture

Components in Underpost follow a consistent pattern:

```javascript
import { ComponentName } from './client/components/core/ComponentName.js';

// Most components expose a Render method or utility functions
const html = await ComponentName.Render(options);
```

## Usage Example

Here's a complete example combining multiple components:

```javascript
import { Modal } from './Modal.js';
import { Input } from './Input.js';
import { BtnIcon } from './BtnIcon.js';
import { Validator } from './Validator.js';
import { Translate } from './Translate.js';

// Render a modal with form inputs
await Modal.Render({
  id: 'user-form',
  title: 'User Information',
  html: async () => html`
    ${await Input.Render({
      id: 'username-input',
      type: 'text',
      placeholder: Translate.Render('username'),
      label: Translate.Render('username')
    })}
    ${await BtnIcon.Render({
      label: html`<i class="fas fa-save"></i> ${Translate.Render('save')}`,
      class: 'btn-submit'
    })}
  `
});

// Set up validation
const validate = Validator.instance([
  {
    id: 'username-input',
    rules: [
      { type: 'isEmpty' },
      { type: 'isLength', options: { min: 2, max: 20 } }
    ]
  }
]);
```
