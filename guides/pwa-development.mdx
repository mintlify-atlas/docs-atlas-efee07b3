## Overview

Underpost provides a complete PWA development environment with built-in support for offline functionality, service workers, and installable web apps. Configure your PWA metadata and views through the centralized configuration system.

## PWA Configuration

Define your PWA metadata in `conf.js` to control how your application appears when installed:

```javascript conf.js
const DefaultConf = {
  client: {
    default: {
      metadata: {
        title: 'PWA Demo App',
        backgroundImage: 'assets/background/white0-min.jpg',
        description: 'Web application',
        keywords: ['web', 'app', 'spa', 'demo', 'github-pages'],
        author: 'https://github.com/underpostnet',
        thumbnail: 'android-chrome-384x384.png',
        themeColor: '#ececec',
        pwaAssetsPath: '',
      },
      // ...
    }
  }
};
```

### Metadata Fields

<ParamField path="title" type="string">
  The application name displayed on the home screen
</ParamField>

<ParamField path="description" type="string">
  Short description of your PWA
</ParamField>

<ParamField path="themeColor" type="string">
  Browser theme color (hex format)
</ParamField>

<ParamField path="thumbnail" type="string">
  Path to app icon (recommended: 384x384px)
</ParamField>

## Configuring Views

Define your application routes with SSR (Server-Side Rendering) support:

```javascript conf.js
views: [
  { path: '/', title: 'Home', client: 'Default', ssr: 'Default' },
  { path: '/home', title: 'Home', client: 'Default', ssr: 'Default' },
  { path: '/settings', client: 'Default', ssr: 'Default' },
  { path: '/log-in', client: 'Default', ssr: 'Default' },
  { path: '/sign-up', client: 'Default', ssr: 'Default' },
  { path: '/account', client: 'Default', ssr: 'Default' },
  { path: '/docs', client: 'Default', ssr: 'Default' },
  { client: 'Default', ssr: 'Default', path: '/404', title: '404 Not Found' },
]
```

<Steps>

<Step title="Define Routes">
  Add route objects to the `views` array with path, title, and rendering configuration
</Step>

<Step title="Configure SSR">
  Specify which SSR template to use for each route (e.g., 'Default')
</Step>

<Step title="Set Client Components">
  Define which client bundle handles the route rendering
</Step>

</Steps>

## SSR Configuration

Configure server-side rendering with custom head and body sections:

```javascript conf.js
ssr: {
  Default: {
    head: ['Seo', 'Pwa', 'Css', 'DefaultScripts', 'Production'],
    body: ['CacheControl', 'DefaultSplashScreen'],
    mailer: { 
      userVerifyEmail: 'DefaultVerifyEmail', 
      userRecoverEmail: 'DefaultRecoverEmail' 
    },
    offline: [
      { 
        path: '/offline', 
        title: 'No Network Connection', 
        client: 'NoNetworkConnection', 
        head: [], 
        body: [] 
      },
      { 
        path: '/maintenance', 
        title: 'Server Maintenance', 
        client: 'Maintenance', 
        head: [], 
        body: [] 
      },
    ],
  },
}
```

### SSR Components

- **Seo**: Meta tags and SEO optimization
- **Pwa**: Progressive Web App manifest and meta tags
- **Css**: Style injection
- **DefaultScripts**: Core JavaScript bundles
- **CacheControl**: Browser caching headers
- **DefaultSplashScreen**: Loading screen while app initializes

## Offline Support

Underpost automatically generates offline pages for network failures:

<CodeGroup>

```javascript Offline Page Config
offline: [
  { 
    path: '/offline', 
    title: 'No Network Connection', 
    client: 'NoNetworkConnection', 
    head: [], 
    body: [] 
  },
]
```

```javascript Service Worker
// Underpost automatically generates service workers
// based on your offlineBuild configuration
server: {
  'default.net': {
    '/': {
      client: 'default',
      offlineBuild: true, // Enable offline support
    }
  }
}
```

</CodeGroup>

## Component Registration

Register client components to be loaded in your PWA:

```javascript conf.js
components: {
  core: [
    'CommonJs',
    'VanillaJs',
    'Responsive',
    'Router',
    'Modal',
    'Auth',
    'SocketIo',
    // ... more components
  ],
  default: [
    'MenuDefault',
    'RoutesDefault',
    'ElementsDefault',
    'CssDefault',
    'LogInDefault',
  ],
}
```

<Note>
  **Core components** provide fundamental framework functionality, while **default components** are your custom application logic.
</Note>

## Distribution Packages

Include third-party libraries in your PWA build:

```javascript conf.js
dists: [
  {
    folder: './node_modules/@neodrag/vanilla/dist/min',
    public_folder: '/dist/@neodrag-vanilla',
    import_name: '@neodrag/vanilla',
    import_name_build: '/dist/@neodrag-vanilla/index.js',
  },
  {
    folder: './node_modules/socket.io/client-dist',
    public_folder: '/dist/socket.io',
    import_name: 'socket.io/client-dist/socket.io.esm.min.js',
    import_name_build: '/dist/socket.io/socket.io.esm.min.js',
  },
]
```

## Build Configuration

Control your PWA build process:

```javascript conf.js
server: {
  'default.net': {
    '/': {
      client: 'default',
      minifyBuild: false,    // Minify JavaScript/CSS
      iconsBuild: false,     // Generate PWA icons
      liteBuild: true,       // Lightweight build
      docsBuild: false,      // Include documentation
      offlineBuild: false,   // Enable offline support
    }
  }
}
```

<Warning>
  Enable `minifyBuild` and `iconsBuild` in production for optimized performance and proper PWA installation.
</Warning>
